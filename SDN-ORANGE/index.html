HOW THE RYU CONTROLLER WORKS

The Ryu controller handles every packet_in event:

1. MAC Learning — source MAC ↔ port mapping stored per switch.
2. Drop Check — if packet matches a drop rule, a flow_mod(actions=[]) is pushed and packet discarded.
3. Forward — if dst MAC known, install forwarding flow (priority=10, idle_timeout=10s); else flood.

Drop flows use empty actions list = OpenFlow DROP. Priority 200 ensures they override forwarding rules (priority=10).

MAC LEARNING SWITCH LOGIC

On every packet_in event the controller reads src_mac and in_port and stores them: mac_table[dpid][src_mac] = in_port. When a packet arrives for a known dst MAC, the controller installs a forwarding flow and sends the packet out the correct port. Unknown dst MACs are flooded to all ports.
